Основные вопросы на грейдировании

HTML

  1. HTML это

    - Язык гипертекстовой разметки, позволяет создавать форматированный текст, который понимает браузер и использует для отрисовки страницы

    Скелет страницы, основная задача объяснить браузеру как отрисовать станицу

  2. Глобальные атрибуты HTML

    • class - позволяет искать и стилизовать элементы
    • contenteditable - делает элемент редактируемым, тоесть даёт возможность изменить контент (булевой)
    • data-* - группа атрибутов позволяющая перекидывать данные между html и DOM, часто используется для управления html элементами через js
    • hidden - скрывает элемент от людей и роботов
    • spellcheck - показывает браузеру что нужно проверить равописание в элементе
    • tabindex - показывает браузеру нужно ли видеть элемент при переключении по странице табами

  3. Категории контента HTML5 это

    - Группировки элементов объединённые общим набором характеристик, не задёт отношения, а помогает описывать поведение и правила

  4. Основные категории контента

    • Метаданные - элементы метаданных изменяют отображение или поведение документа, связывают его с другими докумантами и представляют иную доп. информацию
    • Потоковый - элементы основного потока содержащие текст и встроенный контент
    • Секционный - элементы создающие глобальные секции в структуре документа, определяют область заголовочного контета, хедер и футер
    • Заголовочный - определяют заголовки секций
    • Фразовый - определяет текст и его формат
    • Встроенный - импортирует другой ресурс или вставляет содержимое на другом языке резметки
    • Интерактивный - элементы для взаимодействия с пользователем
    • Явный - то что видно на странице, не скрыто и не пусто
    • Контент форм - элементы принадлежащие формам

  5. DOCTYPE это

    - Используется для указания типа документа, пишется первой строкой любого html, помогает браузеру интерпритировать страницу и понять какие теги валидные, а какие нет

  6. Базовая структура HTML страницы

    • DOCTYPE - html
    • htm - lang="ru"
    • head - meta, title, link...
    • body - контент

  7. Валидация и типы проверки

    - Проверка документа валидатором, на соответствие стандартам и поиск ошибок

    • dtd - соответсвие DOCTYPE
    • Синтаксис
    • Вложенность
    • Наличие посторонних предметов

  8. Какой тег нужен для вёрстки кнопки

    • <button>Button</button> - обычная, для настройки функционала через js
    • <button type="submit"> Button </button> - кнопка подтверждения формы
    • <input type="submit"> value="Button"> - кнопка подтверждения формы
    • <input type="button"> - обычная старая
    • <a href="#">Button</a> - ссылка кнопка, стилизуется через css

  9. Составляющие HTML5 как платформу

    • Сементика - опысывает контент
    • Стилизация - создаёт темы оформления
    • Доступ к устройствам - доступ к устройствам ввода и вывода
    • Связанность - общение с сервером
    • Офлайн хранилище - хранение данных локально, на стороне клиента
    • Мультимедиа - подключение аудио и видео
    • 2D/3D графика - расширяет возможности презентации
    • Производительность и интернет большая скорость оптимизации и лучшее использование аппаратных средств

  10. Инлайновый стиль это

    - Стиль указанный в элементе html документа, через тег style, переопределить можно только через !important

  11. Дефолтные стили у HTML элементов

    Практически все элементы имеют дефолтные стили, например отступы у заголовков, маркеры и цифры у списков, так же существуют теги вроде i или b, которые нужны только для визуально изменения. Разные браузеры отображают дефолтные стили по своему и для того что бы вёрстка была кроссбраузерной - используют нормалайз или ресет дефолтных стилей.

  12. Что такое семантика и какие теги семантические

    Это использование правильных тегов описывающих содержимое внутри себя, тег имеющий смысловое значение. Семантичных тегов очень много, например header, main, footer, nav, p, h1, ul

  13. Как правильно сверстать картинку с подписью

    Внутрь тега figure положить картинку img, добавить туда же подпись с помощью figcaption, пояснение может быть как для 1 картинки так и для нескольких внутри figure. Кроме семантики этот подход делает сайт доступнее, так как скринридер будет зачитывать подпись отмечая что она относится к карттинке.

  14. Типы списков в HTML

    Маркированный ul li, нумерованный ol li, список определений dl (обёртка) dt (заголовок) dd (само определение)

  15. Атрибут alt, и зачем он нужен

    Атрибут alt (альтернативный текст) добавляется для картинки img, если картинка не отобразится при загрузке страницы - будет показан текст alt. Это обязательный атрибут, валидатор не пропустит без него. Так же улучшает доступность страницы, так как скринридер зачитывает содержимое alt.

  16. Разница между strong, em и b, i

    strong и em - семантические теги, b и i - только для визуального изменения. При использовании первых слово будет не только выделено, но и при чтении скринридер сделает на нём акцент.

  17. Типы input элементов

    input элемент нужен для получения вводимой информации, имеет атрибут type, взависимости от которого будет применён тот или иной тип

    • text- самый распростронённый, для ввода букв, цифр и символов
    • password - для паролей, вводимые символы скрыты звёздочками
    • email - для ввода почтового адреса, при фокусировке с мобилки, клавиатура будет содержать символы ускоряющие ввод email
    • number - позволяет вводить только цифры, при фокусировке с мобилки, откроется клавиатура содержащая только цифры
    • button и submit - в виде кнопки, submit позволяет отправить форму
    • checkbox и radio - заменяет поле ввода на квадрат с галочкой или кружок с точкой соответственно
    • date, datetime, datetime-local - для ввода даты, могут содержать кнопки или выпадающий календарь

  18. Data-атрибуты

    Немного устаревший подход, применяется для хранения информации, которую вдальнейшем можно использовать в js для манипуляций. В DOM можно хранить доп данные, не прибегая к нестандартным решениям. Минус - не безопасно, так как изменить атрибут можно прям в консоли разработчика

  19. Script, script async и script defer

    • script - тег для хранения исполняемого js кода, блокирует чтение документа до момента окончания исполнения js, принято добавлять в конец документа
    • script async1 - исполняется паралельно с чтением html документа, применяется к скриптам независящим от других
    • script defer1 - извлечёт код при чтении страницы, но выполнится только после полного его парсинга, если таких кодов несколько, то они будут выполняться по порядку
    • (1 - атрибут используется для скриптов в которых предусмотрена работа с DOM элементами)

  20. Datalist

    Нужен для реализации автозаполнения выпадающего списка, указывается input с атрибутом list='пример', после указывается datalist с id='пример' (что связывает его со списком), внутри указываeтся несколько option c атрибутом value='название', которые предлагаются для автозаполнения инпута при клике, но так же остаётся возможность ввести своё значение, не выбрав ниодного из предложеных

  21. Почему link подключают в head, а script перед закрытием body

    • Помещение link в head прописано в спецификации
    • Страница читается слева на право с верху вниз и стили ставят вверх, что бы они как можно быстрее их запросить и подключить
    • Если поместить стили вниз браузер может блокировать загрузку страницы, чтобы избежать перерисовки документа, и пользователь будет видеть белую страницу пока всё не подгрузится
    • script встречающийся в коде - тормозит построемие DOM и стилизацию, пока сам не будет выполнен (можно избежать атрибутами асинг и дефер)
    • Часто скрипту требуются DOM элементы, поэтому его ставят вниз для того что бы быть уверенным что всё что нужно для работы прогрузилось

  22. Meta теги

    Специальные теги внутри head, для описания страницы поисковым системам, содержат дополнительную информацию для роботов и нормальной работы на разных девайсах, не отображаются на странице

  23. Что описывается в head

    • title - заголовок страницы
    • meta - кодировка, описание + ключи, вспомогательные механизмы для браузеров
    • link - пути для стилей, шрифты, иконка
    • script - скрипты через src или на прямую

  24. Тэги tr, th, td

    Используются в таблице, не самостоятельные

    • tr - создаёт строку таблицы
    • th - ячейка заголовка (жирный), распологается в строке
    • td - простая ячейка, распологается в строке

  25. Meta-тег viewport

    Для построения отзывчивой версии сайта, в свойстве "content" описывается как страница должна себя вести (Ширина страницы по ширине экрана, масштабирование при первом открытии).

  26. Сanvas это

    Это элемент "холст" в который можно вставлять сложные анимации, изображения, градиенты. Через js можно рисовать объекты и изменять их свойства. Элемент для отрисовки графики

  27. Svg и canvas это

    • svg - язык разметки масштабированной векторной графики, для написания 2d графики в формате xml, поддерживает неподвижную и анимированную графику
    • canvas - элемент для рисования графики через js, используется для колладжей, графиков, анимаций

  28. Разница svg и canvas

    • svg - набор элементов, язык разметки, для векторной графики, есть свой формат (можно сохранить отдельно с соотв расширением), сжимается
    • canvas - одиночный DOM элемент, скриптовый язык, для растровой графики, нет своего формата (нельзя сохраниить в отдельный файл), не сжимается

  29. Когда использовать svg и canvas

    • svg - создание пользовательских интерфейсов независимых от разрешения, высокоитерактивных анимированных интерфейсов, создание диаграмм, редактирование векторных изображений
    • canvas - для редактирования изображений, обрезки, фильтров и тд, создание графиков функций, создание гистограм, создание игровой графики - спрайты и фоны

  30. Плюсы и минусы svg и canvas

    • svg -
      • + Не зависит от разрешения экрана, отличная анимация описанием или js, полный контроль над DOM элементами, хранится в формате XML
      • - Низкая скорость при более сложном строении, плохо подходит для игр
    • canvas -
      • + высокая стабильная производительность, полученые изображения можно сохранять в png или jpg, подходит для графики в пикселях
      • - Отрисовка на пикселях, слабые возможности для текста

  31. Атрибут autocomplete?

    Позволяет заполнить поле текстом введённым ранее, значение сохраняется и предлагается в следующий раз. В целях безопасности может быть отключено пользователем или разработчиком для запрета сохранения пароля

  32. Элемент output

    Позволяет отобразить сумму 2х чисел из формы в виде текста.

  33. Свойство valueAsNumber

    Свойство для числовых полей формы, возвращает значение ввиде числа, а не строки, данные сумммируются и вычисляются без приведения к числовому типу

  34. Атрибут target

    Показывает как стоить открыть ссылку, по умолчанию открывает в текущем окне, все значения начинаются с _

    • _blank - откроет в новом окне
    • _self - в текущее окно
    • _parent - во фрейм родитель, если их нет то как self
    • _top - отменяет все фреймы и загружает в полном окне, если их нет то как self

  35. ApplicationCache

    Позволяет указать браузеру какие файлы нужно кешировать и сделать доступными для оффлайн просмотра. Даёт больше скорости и меньше нагрузки на сервер, так как часть информации уже загружена.

  36. Элемент picture

    Испотльзуется когда для разных устройств или вариантов отображения нужны разные картинки. Это контейнер для 1 элемента img и нескольких запасных source (например несколько вариантов для разной ширины жкрана).

  37. Атрибут srcset

    Является атрибутом img, пишется после основного пути src, предлагает браузеру несколько изображений отображения на разных разрешениях и учитывания высокое разрешение retina

  38. Навигационное меню

    nav - ul - li - a

  39. Элемент iframe

    Контейнер внутри документа, позволяет вставить сторонний html документ на страницу. Обычно является итерактивным - карта, видео, комменты с соц сети. Существует независимо от документа. Стилизации поддаётся только контейнер, но не содержимое.

  40. Теги sup и sub

    Текстовые теги, для написания чего-либо в верхнем или нижнем регистре. Можно вкладывать друг в друга для создания сложгых формул

  41. Как скрыть элемент без CSS и JS?

    Атрибут hidden в html разметке (работает как display:none). Плохая практика

  42. Элементы meter и progress

    • meter - числовое значение в известном диапазоне, используется только когда известно максимальное значение (например отображение оставшегося места на диске), имеет различные цветовые обозначения в зависимости от наполнености шкалы
    • progress - отобаржает ход выполнения задачи, можно использовать не зная конечного значения

  43. Группировака в select

    select это выпадаюший список содержаший опции, каждая опция обёрнута в option. Для групировки можно использовать optgroup (Можно добавить lebel для добавления заглавия 1 или нескольких групп). Нельзя вкладывать optgroup друг в друга. Существует атрибут "дизаблет" делающий опции внутри своей группы недоступными для выбора

  44. Скринридер

    Это устройство читающее страницу (или передающее её шрифтом браиля). В основном используется слабовидящими людьми или имеющими отклонение в восприятии статичной информации.

  45. Как изменить форму картинки или элемента

    свойство clip-path - создаёт ограниченую область определяя какая часть картинки должна быть видна. Обрезаня область это форма определённая внутренней ссылкой, внешним svg или как фигура. Может использоваться в css (что позволяет применять фигуры и легко использовать) или на svg - можно создать фигуру и обрезать по ней элемент с помощью специального синтаксиса (преимуществом является лучшая поддержка и возможность создавать более сложные фигуры)

  46. Отличается article от section

    Артикль- независимый самостоятельный элемент, который может быть убран без ущерба для страницы

    Секция- используется как контейнер, внутренние объекты объеденины общим смыслом, задаёт раздел документа, обычно содержит заголовок, можно вкладывать друг в друга

  47. Стилизация svg

    Основана на XML, основные атрибуты - fill (заливка) и stroke (обводка). На их базе строятся остальные свойства вроде прозрачности и тд. Можно стилизовать напрямую в разметке, или через css. Если используются спрайты то стилизовать можно только в разметке.

  48. Что такое ARIA роли в веб приложении

    АРИА - это большой блок доступности, набор атрибутов для увеличения доступности людям с огр возможностими.

    Атрибут role определяет роли объекта (статья, меню, картинка). Например сообщения об ошибках, навигационные маркеры.

  49. Кнопка и ссылка

    • Ссылка - элемент навигации. Обычно представляет собой текст и служит для перехода на другую страницу или другой раздел страницы
    • Кнопка - функциональный документ. Она отвечает за выполнение определённой функции на странице: открыть, купить, отправить...

      • Если обернуть адрес кнопкой, то её нельзя будет открыть в новом окне и через сочетание клавишь ctr + лкм

      Если функционал ссылкой, то нажатие пкм открывает дефолтное меню, но не имеет смысла, так как не может сохранить адрес и тд.

      Принаведении на ссылку - её адрес отображается внизу окна просмотра

      У кнопки нет состояния :visited (посещённые)

      У ссылки нет состояния :disabled (неактивные)

  50. Атрибут decoding

    Перед показом картинки браузер её декодирует - превращает в набор пикселей, что занимает определённое время.

    Атрибут декодинг говорит браузеру как быстро стоит отрисовать картинку в зависимости от её указанной потребности.

    • async — декодировать асинхронно (можно попозже, когда получится)
    • sync — синхронно (нужно прямо сейчас)
    • auto — автоматически (браузер, решай сам)

  51. Атрибут enterkeyhint

    Атрибут указывает, какую метку действия представить для клавиши enter на виртуальных клавиатурах.

    • enter - вставляется новая строка
    • done - означает, что больше нечего вводить, и редактор будет закрыт
    • go - переходит к целевому тексту, который он ввел
    • next - переходит к следующему полю, которое будет принимать текст
    • previous - переходит к предыдущему полю, которое будет принимать текст
    • search - переходит к результатам поиска текста, который он ввел
    • send - текст отправляется адресату

  52. Атрибут novalidate

    Для отключения нативной валидации на стороне браузера. Говорит браузеру не проверять поля и убрать всплывающие подсказки (для показа собственных)

  53. Атрибут inputmode

    Подсказывает браузеру (в отличии от type который указывает явно) на устройствах с экранной клавиатурой какой набор клавиш предложить. Используется на input и textaria

  54. Атрибут pattern

    Определяет регулярное выражение, которому должно соответствовать значение введённое в форму. Если не соответствует - поле подсвечивается как невалидное. форма не отправится пока поле не будет заполнено правильно

  55. Почему стоит использовать семантические теги

    Семантическая вёрстка — подход к разметке, который опирается не на содержание сайта, а на смысловое предназначение каждого блока и логическую структуру документа.

    • Делает сайт доступным - скринридер понимает семантические теги и делает акцент в нужных местах страницы или например может предложить навигацию по заголовкам
    • Поднимает сайт в поисковиках - семантическая разметка помогает роботамлучше понимать, что находится на странице
    • Это соответствует стандартам

  56. Тег label

    устанавливает связь между определенной меткой, в качестве которой обычно выступает текст, и элементом формы input, select, textarea. Такая связь необходима, чтобы изменять значения элементов формы при нажатии курсором мыши на текст. Кроме того, с помощью label можно устанавливать горячие клавиши на клавиатуре и переходить на активный элемент подобно ссылкам

    Существует два способа связывания объекта и метки. Первый заключается в использовании идентификатора id внутри элемента формы и указании его имени в качестве атрибута for тега label. При втором способе элемент формы помещается внутрь контейнера label.
