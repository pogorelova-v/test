Основные вопросы на грейдировании

HTML

  1. HTML это

    - Язык гипертекстовой разметки, позволяет создавать форматированный текст, который понимает браузер и использует для отрисовки страницы

    Скелет страницы, основная задача объяснить браузеру как отрисовать станицу

  2. Глобальные атрибуты HTML

    • class - позволяет искать и стилизовать элементы
    • contenteditable - делает элемент редактируемым, тоесть даёт возможность изменить контент (булевой)
    • data-* - группа атрибутов позволяющая перекидывать данные между html и DOM, часто используется для управления html элементами через js
    • hidden - скрывает элемент от людей и роботов
    • spellcheck - показывает браузеру что нужно проверить равописание в элементе
    • tabindex - показывает браузеру нужно ли видеть элемент при переключении по странице табами

  3. Категории контента HTML5 это

    - Группировки элементов объединённые общим набором характеристик, не задёт отношения, а помогает описывать поведение и правила

  4. Основные категории контента

    • Метаданные - элементы метаданных изменяют отображение или поведение документа, связывают его с другими докумантами и представляют иную доп. информацию
    • Потоковый - элементы основного потока содержащие текст и встроенный контент
    • Секционный - элементы создающие глобальные секции в структуре документа, определяют область заголовочного контета, хедер и футер
    • Заголовочный - определяют заголовки секций
    • Фразовый - определяет текст и его формат
    • Встроенный - импортирует другой ресурс или вставляет содержимое на другом языке резметки
    • Интерактивный - элементы для взаимодействия с пользователем
    • Явный - то что видно на странице, не скрыто и не пусто
    • Контент форм - элементы принадлежащие формам

  5. DOCTYPE это

    - Используется для указания типа документа, пишется первой строкой любого html, помогает браузеру интерпритировать страницу и понять какие теги валидные, а какие нет

  6. Базовая структура HTML страницы

    • DOCTYPE - html
    • htm - lang="ru"
    • head - meta, title, link...
    • body - контент

  7. Валидация и типы проверки

    - Проверка документа валидатором, на соответствие стандартам и поиск ошибок

    • dtd - соответсвие DOCTYPE
    • Синтаксис
    • Вложенность
    • Наличие посторонних предметов

  8. Какой тег нужен для вёрстки кнопки

    • <button>Button</button> - обычная, для настройки функционала через js
    • <button type="submit"> Button </button> - кнопка подтверждения формы
    • <input type="submit"> value="Button"> - кнопка подтверждения формы
    • <input type="button"> - обычная старая
    • <a href="#">Button</a> - ссылка кнопка, стилизуется через css

  9. Составляющие HTML5 как платформу

    • Сементика - опысывает контент
    • Стилизация - создаёт темы оформления
    • Доступ к устройствам - доступ к устройствам ввода и вывода
    • Связанность - общение с сервером
    • Офлайн хранилище - хранение данных локально, на стороне клиента
    • Мультимедиа - подключение аудио и видео
    • 2D/3D графика - расширяет возможности презентации
    • Производительность и интернет большая скорость оптимизации и лучшее использование аппаратных средств

  10. Инлайновый стиль это

    - Стиль указанный в элементе html документа, через тег style, переопределить можно только через !important

  11. Дефолтные стили у HTML элементов

    Практически все элементы имеют дефолтные стили, например отступы у заголовков, маркеры и цифры у списков, так же существуют теги вроде i или b, которые нужны только для визуально изменения. Разные браузеры отображают дефолтные стили по своему и для того что бы вёрстка была кроссбраузерной - используют нормалайз или ресет дефолтных стилей.

  12. Что такое семантика и какие теги семантические

    Это использование правильных тегов описывающих содержимое внутри себя, тег имеющий смысловое значение. Семантичных тегов очень много, например header, main, footer, nav, p, h1, ul

  13. Как правильно сверстать картинку с подписью

    Внутрь тега figure положить картинку img, добавить туда же подпись с помощью figcaption, пояснение может быть как для 1 картинки так и для нескольких внутри figure. Кроме семантики этот подход делает сайт доступнее, так как скринридер будет зачитывать подпись отмечая что она относится к карттинке.

  14. Типы списков в HTML

    Маркированный ul li, нумерованный ol li, список определений dl (обёртка) dt (заголовок) dd (само определение)

  15. Атрибут alt, и зачем он нужен

    Атрибут alt (альтернативный текст) добавляется для картинки img, если картинка не отобразится при загрузке страницы - будет показан текст alt. Это обязательный атрибут, валидатор не пропустит без него. Так же улучшает доступность страницы, так как скринридер зачитывает содержимое alt.

  16. Разница между strong, em и b, i

    strong и em - семантические теги, b и i - только для визуального изменения. При использовании первых слово будет не только выделено, но и при чтении скринридер сделает на нём акцент.

  17. Типы input элементов

    input элемент нужен для получения вводимой информации, имеет атрибут type, взависимости от которого будет применён тот или иной тип

    • text- самый распростронённый, для ввода букв, цифр и символов
    • password - для паролей, вводимые символы скрыты звёздочками
    • email - для ввода почтового адреса, при фокусировке с мобилки, клавиатура будет содержать символы ускоряющие ввод email
    • number - позволяет вводить только цифры, при фокусировке с мобилки, откроется клавиатура содержащая только цифры
    • button и submit - в виде кнопки, submit позволяет отправить форму
    • checkbox и radio - заменяет поле ввода на квадрат с галочкой или кружок с точкой соответственно
    • date, datetime, datetime-local - для ввода даты, могут содержать кнопки или выпадающий календарь

  18. Data-атрибуты

    Немного устаревший подход, применяется для хранения информации, которую вдальнейшем можно использовать в js для манипуляций. В DOM можно хранить доп данные, не прибегая к нестандартным решениям. Минус - не безопасно, так как изменить атрибут можно прям в консоли разработчика

  19. Script, script async и script defer

    • script - тег для хранения исполняемого js кода, блокирует чтение документа до момента окончания исполнения js, принято добавлять в конец документа
    • script async1 - исполняется паралельно с чтением html документа, применяется к скриптам независящим от других
    • script defer1 - извлечёт код при чтении страницы, но выполнится только после полного его парсинга, если таких кодов несколько, то они будут выполняться по порядку
    • (1 - атрибут используется для скриптов в которых предусмотрена работа с DOM элементами)

  20. Datalist

    Нужен для реализации автозаполнения выпадающего списка, указывается input с атрибутом list='пример', после указывается datalist с id='пример' (что связывает его со списком), внутри указываeтся несколько option c атрибутом value='название', которые предлагаются для автозаполнения инпута при клике, но так же остаётся возможность ввести своё значение, не выбрав ниодного из предложеных
